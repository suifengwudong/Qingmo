# [Feature] 文件树重命名与删除操作

## 描述
当前文件树缺少重命名（`F2` 快捷键）和删除（移入废稿/永久删除）两项基础操作，用户只能通过外部文件管理器修改项目结构，体验割裂。

## 期望行为
1. **重命名**：在文件树中选中文件/文件夹后按 `F2`，或通过右键菜单选择「重命名」，弹出内联输入框，确认后将磁盘文件名与 UI 显示同步更新。
2. **删除**：右键菜单提供「移入废稿文件夹」（逻辑删除，文件移至项目根目录 `废稿/` 子目录）和「永久删除」（物理删除）两个选项；执行前弹出确认对话框，避免误操作。
3. **状态同步**：重命名/删除后自动刷新文件树（`refresh_tree()`）并清除对应编辑器标签页，防止引用失效。

## 实现建议

### 重命名流程
```
用户按 F2 / 右键「重命名」
  → 将选中节点切换为 inline edit 状态（TextEdit）
  → 用户输入新名称后按 Enter 确认 / Esc 取消
  → 调用 std::fs::rename(old_path, new_path)
  → refresh_tree(); 更新 open_files 中的路径引用
```

### 删除流程
```
用户右键「移入废稿」/「永久删除」
  → 弹出 egui Modal 确认对话框
  → 「移入废稿」: std::fs::rename(path, project_root/废稿/filename)
  → 「永久删除」: std::fs::remove_file(path) 或 remove_dir_all(path)
  → refresh_tree(); 关闭对应标签页
```

### 相关文件
- `src/app/mod.rs`：文件树渲染与右键菜单（搜索 `context_menu` 相关代码）
- `src/app/file_manager.rs`：新增 `rename_file`, `delete_file`, `move_to_draft` 方法
- `src/app/sync.rs`：`refresh_tree()` 已存在，可直接调用

## 优先级
🔴 高——基础文件操作缺失严重影响写作流程

## 验收标准
- [ ] 文件/文件夹可通过 F2 或右键菜单重命名，磁盘与 UI 保持一致
- [ ] 文件/文件夹可通过右键菜单删除，删除前有确认弹窗
- [ ] 删除后对应编辑器标签页自动关闭
- [ ] 重命名/删除操作失败时状态栏显示错误信息
