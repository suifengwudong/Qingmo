# [优化] 全文搜索升级：正则表达式支持与结果预览

## 描述
当前全文搜索（`search_dir`）仅支持固定字符串匹配，对于作者常见的需求（如"查找所有对话行"、"查找所有出场人物描写"）功能有限。升级搜索支持正则表达式，并改善结果预览体验。

## 期望行为
- 搜索框旁增加「正则」开关（`use_regex: bool`），默认关闭
- 开启后，搜索词被解析为正则表达式（使用标准库 `regex` crate）；语法错误时状态栏提示
- 搜索结果每条展示：文件名 + 行号 + **带高亮的上下文片段**（匹配词加粗或着色）
- 结果窗口支持键盘导航（↑↓选择，Enter 打开对应文件并跳转到该行）
- 「大小写敏感」开关与正则模式互通（正则模式下映射为 `(?i)` 前缀）

## 实现建议
- `Cargo.toml` 新增依赖：`regex = "1"（features = ["std"]）`
- `search.rs` 的 `search_dir` 函数增加 `use_regex: bool` 参数，内部 `if use_regex { Regex::new(query)? } else { 精确匹配 }`
- `SearchResult` 结构体增加 `line_no: usize`、`context: String` 字段
- UI 跳转逻辑：打开文件后，通过 `TextEdit` 的 `id` + egui cursor API 定位到对应行

## 优先级
🟡 中

## 验收标准
- [ ] 固定字符串搜索结果与升级前完全一致（无回归）
- [ ] 正则模式下 `"第[一二三]章"` 能匹配 "第一章"、"第三章"，不匹配 "第四章"
- [ ] 非法正则表达式输入时状态栏显示友好错误，不 panic
- [ ] `search_dir` 新增参数有单元测试（正则匹配 + 结果行号验证）
